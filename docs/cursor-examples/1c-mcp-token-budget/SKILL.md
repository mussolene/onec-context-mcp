# MCP 1c-help: токен-бюджет и порядок вызовов

## Цель

Меньше текста в контексте при том же качестве: короткие ответы справки, меньше «шума» (WebЦвета и т.п. на технических запросах), меньше повторных обходов.

## Порядок инструментов (сначала дешёвое и точное)

1. **Метаданные конфигурации** — если вопрос про объект УФ/ОФ, регистр, документ: `search_1c_metadata_exact` → `get_1c_metadata_object(object_id)`. Для полей: `search_1c_metadata_fields(object_query=..., field_query=...)` — оба параметра обязательны (id или имя объекта + фрагмент имени поля).
2. **Точное имя API платформы** — `get_1c_api_answer("Тип.Метод")` с **`detail` по умолчанию (compact)**; `detail="full"` только если не хватает параметров, синтаксиса, ограничений.
3. **Структура типа** — `get_1c_api_object("ИмяТипа")` при необходимости truth-source без простыни текста.
4. **Широкий подбор по платформе** — `search_1c_api(query)` — один-два осмысленных запроса; не дублировать синонимами подряд.
5. **Естественный язык** — `answer_1c_help_question(question)` только если нет кандидата `Тип.Метод` и нужна сводка по теме; при ответе «не уверенно» уточнить запрос или перейти к п. 2–4.
6. **Стандарты / сниппеты** — `search_1c_standards` / `search_1c_snippets` только когда это явно нужно задаче (не «на всякий случай»).

## Не делать

- Не передавать **простыню** в `get_1c_api_answer` — сработает отказ и подсказка; использовать `answer_1c_help_question` или `search_1c_api`.
- Не гонять **параллельно 8+** тяжёлых вызовов 1c-help: суммарный объём ответов всё равно попадает в контекст; лучше 3–5 целевых запросов.
- Не запрашивать `detail="full"` до первого **compact**-ответа, если задача — найти сигнатуру или факт.

## L2 (сессия): одна строка на повторяющиеся темы

Примеры шпаргалок, чтобы не перечитывать справку:

- СКД → таблица значений: `КомпоновщикМакетаКомпоновкиДанных.Выполнить(..., Тип("ГенераторМакетаКомпоновкиДанныхДляКоллекцииЗначений"))` → `ПроцессорКомпоновкиДанных` → `ПроцессорВыводаРезультатаКомпоновкиДанныхВКоллекциюЗначений.Вывести`.
- Параметры периода в настройках: `НастройкиКомпоновкиДанных.ПараметрыДанных.УстановитьЗначениеПараметра`.

## Поведение MCP 1c-help в этом репозитории

В `search_1c_api` для технических запросов отфильтрованы типичные нерелевантные хиты (например `WebЦвета.*`, `БиблиотекаКартинок.*`). Для вопросов про СКД ответы от явно нерелевантных API (например `ТаблицаФормы`) не подставляются вместо компоновки.

## Связанные скиллы

- **1c-mcp-development** — полный tiered toolset и циклы.
- **three-level-memory** — не повторять чтение файлов и лишние вызовы в сессии.
- **token-efficient** — маршрутизация простого boilerplate на локальную модель, где уместно.
