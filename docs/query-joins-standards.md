# Несколько внутренних соединений в запросе и стандарты 1С

## Можно ли использовать несколько ВНУТРЕННЕЕ СОЕДИНИЕ в одном запросе?

**Да.** Язык запросов 1С допускает несколько соединений подряд в одном запросе. Синтаксис:

```bsl
ВЫБРАТЬ
    Таблица1.Поле1,
    Таблица2.Поле2,
    Таблица3.Поле3
ИЗ
    Справочник.Таблица1 КАК Таблица1
    ВНУТРЕННЕЕ СОЕДИНЕНИЕ Справочник.Таблица2 КАК Таблица2
    ПО Таблица1.Ссылка = Таблица2.Владелец
    ВНУТРЕННЕЕ СОЕДИНЕНИЕ Документ.Таблица3 КАК Таблица3
    ПО Таблица2.Ссылка = Таблица3.Контрагент
```

Слово «ВНУТРЕННЕЕ» можно опускать (достаточно `СОЕДИНЕНИЕ`).

## Что говорят стандарты (v8std, ITS)

Стандарты **не запрещают** несколько внутренних соединений с обычными таблицами. Они ограничивают **что** можно соединять и **как** задавать условие:

| Стандарт | Суть |
|----------|------|
| **v8std: Ограничения на соединения с вложенными запросами и виртуальными таблицами** (ст. 655) | Не рекомендуется использовать в качестве правой части соединения вложенные запросы и виртуальные таблицы — это может приводить к плохой производительности и проблемам на части СУБД. |
| **v8std: Ограничения на использование вложенных запросов в условии соединения** (ст. 656) | Не следует использовать вложенные запросы **в условии** соединения (в части ПО …). Это может привести к значительному замедлению и на некоторых СУБД — к полной неработоспособности. Рекомендуется переписать запрос с **временными таблицами**. |
| **v8std: Ограничение на использование конструкции "ПОЛНОЕ ВНЕШНЕЕ СОЕДИНЕНИЕ"** (ст. 435) | Отдельные ограничения на использование полного внешнего соединения в запросах. |

Итого: **несколько внутренних соединений по «плоским» таблицам (справочники, документы, таблицы регистров) — нормальная и разрешённая практика.** Избегать нужно:

- соединения **с** вложенным запросом или виртуальной таблицей как с одной из сторон;
- вложенных запросов **в условии** соединения (ПО …); в таких случаях лучше временные таблицы.
