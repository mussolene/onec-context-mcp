# Архив Документации

Здесь лежат исторические материалы: планы, исследования, verification notes, troubleshooting writeups и security audit artifacts.

Эти документы сохранены как decision log и справка для будущих изменений, но не нужны для первого запуска и не входят в основной маршрут onboarding.

## Что находится в архиве

- `plan-*` и implementation plans
- `*-analysis*` и `*-report*`
- verification и test writeups
- migration и troubleshooting notes
- security audit материалы в [security-audit/](security-audit/)

## Когда сюда идти

- нужно восстановить историю решения;
- нужно поднять старый анализ или аудит;
- нужно проверить, почему была выбрана текущая архитектура или workflow;
- нужно найти редкий troubleshooting case, который не должен мешать основному quick start.

Основной маршрут документации: [../README.md](../README.md).
