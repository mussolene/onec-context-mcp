# Стандарты и память MCP (onec_help_memory)

## Где что лежит

| Коллекция Qdrant | Содержимое | Использование |
|------------------|------------|---------------|
| **onec_help** | Справка 1С (распакованная из .hbk) | Семантический и keyword-поиск по справке |
| **onec_help_memory** | Сниппеты, community_help, **стандарты** (v8std, ITS, репо) | `get_1c_code_answer(include_memory=True)`, `search_1c_help(include_user_memory=True)` |

Стандарты загружаются в **ту же коллекцию**, что и сниппеты (`onec_help_memory`), с полем `domain=standards`. Справка остаётся в отдельной коллекции `onec_help`.

## Зачем одна коллекция для сниппетов и стандартов

- Один семантический поиск по «долгосрочной памяти»: в одном запросе можно подтянуть и примеры кода (snippets), и правила/стандарты (standards).
- В ответах MCP источник помечается: `[пример]`, `[инструкция]`, `[стандарт]` — по полю `domain`.
- При необходимости поиск по памяти можно ограничить доменом (например, только `domain=standards`) в коде; в текущих инструментах поиск идёт по всей памяти без фильтра по домену.

## Релевантность и связь со справкой

- **Справка 1С** — описание API, синтаксис, типы, методы. Отвечает на «как вызвать», «что возвращает».
- **Стандарты (v8std, ITS, репо)** — соглашения, практики, ограничения. Отвечают на «как правильно», «чего избегать», «в каком обработчике что делать».

Они дополняют друг друга: по одному запросу `get_1c_code_answer` может вернуть и фрагмент справки (из `onec_help`), и пункт стандарта (из `onec_help_memory`), что даёт и синтаксис, и рекомендации в одном ответе.

## Как проверить, что стандарты попали в индекс

1. **Один и тот же Qdrant для MCP и load-standards**  
   MCP и команда `load-standards` должны ходить в **один и тот же** Qdrant (те же `QDRANT_HOST` и `QDRANT_PORT`). Иначе данные попадут в одну БД, а запросы — в другую.
   - **Docker:** и MCP, и `make load-standards` используют `QDRANT_HOST=qdrant` (сервис в compose). Так и должно быть.
   - **Локальный запуск:** если MCP в Docker, а `load-standards` запускали локально — по умолчанию локальный скрипт подключается к `localhost:6333`. Порт 6333 с хоста проброшен в контейнер Qdrant, так что это один инстанс. Если же у вас Qdrant только в Docker, то и `load-standards` нужно запускать в Docker: `make load-standards ARGS="--its-v8std"`.

2. **Статус индекса**  
   Вызвать MCP `get_1c_help_index_status`. В ответе должна быть строка вида:
   ```text
   Memory (**onec_help_memory**): **N** points (snippets + standards)
   ```
   Если после `load-standards` N не вырос — загрузка в память не выполнялась или не дошла до этого Qdrant (другой хост/порт).

2. **Поиск по формулировкам из стандартов**  
   - `get_1c_code_answer("обработчик ПередЗаписью проверка ОбменДанными.Загрузка", include_memory=True)`  
   - `search_1c_help("стандарт разработки соглашения именование", include_user_memory=True)`  

   В блоке «Из памяти» должны появляться пункты с меткой `[стандарт]` и ссылкой на its.1c.ru или репо.

3. **Достаточность информации**  
   В payload стандартов уходят `title`, `description`, `code_snippet` (тело статьи в markdown). Этого хватает для эмбеддинга и выдачи в ответе. Если нужны только стандарты без справки — можно в коде вызывать `search_long(..., domain="standards")` и выводить только их.

## Загрузка стандартов в память

- **С fetch ITS:** `make load-standards ARGS="--its-v8std"` (Docker) или `python -m onec_help load-standards --its-v8std` (локально). Стандарты качаются с its.1c.ru, пишутся на диск и отправляются в `onec_help_memory`.
- **Только с диска:** если папка `data/standards/its-v8std` уже есть (после прошлого fetch), можно запустить `make load-standards` **без** `--its-v8std` — тогда репо (v8-code-style, v8std) и все `.md` из `its-v8std` подгрузятся в память без повторного запроса к ITS. Удобно, если эмбеддинги уже гонялись (LM Studio в логах), но запись в Qdrant не прошла или коллекцию потом сбросили (reinit).

Стандарты пишутся на диск в `STANDARDS_DIR` (например `data/standards/its-v8std/...`) и при успешном завершении индексируются в `onec_help_memory` с `domain=standards`.
