# Отчёт по релевантности и полноте ответов MCP 1c-help

Шаблон для фиксации результатов проверки качества выдачи MCP: релевантность (топ результатов по запросу) и полнота (достаточность для получения рабочего кода или ответа на вопрос).

## Как заполнять

1. Запустить MCP и Qdrant (`make up` или `make ingest-up`).
2. Для каждого запроса из чек-листа выполнить:
   - `get_1c_code_answer(query)` и при необходимости цепочку `search_1c_help_keyword` → `get_1c_help_topic(topic_path)`.
   - Оценить релевантность: топ-1 или топ-3 содержат нужную тему (да/нет + кратко).
   - Оценить полноту: по выданным топикам можно собрать рабочий пример или ответ (да/нет).
3. При нерелевантности `get_1c_code_answer` проверить наличие подсказки про `search_1c_help_keyword` (см. [quality-and-pitfalls-analysis.md](quality-and-pitfalls-analysis.md)).

## Критерии

- **Релевантность:** в ответе есть путь к теме по запрошенному API/сценарию (криптография, СКД и т.д.) или явная отсылка к нужному разделу справки.
- **Полнота:** в ответе есть синтаксис/пример кода или однозначная ссылка на топик с примером; по ним можно написать рабочий код.

## Регрессия (опционально)

Для выборочных запросов можно сохранять эталонные фрагменты ответа и при повторном прогоне сравнивать наличие ключевых маркеров (путь к топику, имена API). Формат: запрос → список обязательных подстрок в ответе.
